Setup del equipo nuevo

Correr una sola vez después de la migración a Windows.

# 1. Ir a la carpeta de código
cd "C:\Users\IMO Creativo\Documents\Claude\Projects\gambeta-ai\code"

# 2. Autenticar con GitHub CLI (recomendado — evita PAT expuesto)
gh auth login
# Elegir: GitHub.com → HTTPS → Login with a web browser

# 3. Clonar el repo
gh repo clone maurounion10-lab/ia-apuestas .

# 4. Configurar identidad (una vez por máquina)
git config --global user.name "Mauro"
git config --global user.email "pronosticosarg@gmail.com"

# 5. Verificar
git remote -v
git status

Alternativa sin GitHub CLI: usar Git Credential Manager (viene con Git for Windows).

git clone https://github.com/maurounion10-lab/ia-apuestas.git .
# En el primer push te pide login por navegador. Lo guarda en Credential Manager.

Flujo diario

Ver qué está pasando
git status
git diff
git log --oneline -20
Hacer un cambio y subirlo
git pull origin main
# ...editar archivos...
git add .
git commit -m "feat: descripción clara"
git push origin main

Convención de commits

Usar prefijos Conventional Commits:

feat:Feature nueva
fix:Bug fix
docs:Cambios de documentación
style:Formato / CSS / no cambio de lógica
refactor:Refactor sin cambio de comportamiento
perf:Mejora de performance
test:Agregar / arreglar tests
chore:Mantenimiento (deps, config)
content:Post SEO nuevo o edición de copy

Ejemplos buenos

  • feat: sistema de daily reward de A-COIN con cooldown 24h
  • fix: OAuth Google no persistía sesión en Firefox
  • content: post "cuotas campeón Mundial 2026" para mercado México
  • style: nueva paleta oscura + tipografía display en landing

Reglas duras

  1. Nunca pushear .env — está en .gitignore del proyecto. Verificar con git status.
  2. Nunca hardcodear secretos nuevos — usar .env + process.env.X.
  3. Pull antes de push — evita conflictos.
  4. Un push = un cambio lógico — no mezclar features distintas.
  5. PATs de GitHub nunca en archivos del repo — solo en Credential Manager.

Emergencias de git

Rollback de deploy
git revert HEAD + push, o rollback desde Cloudflare dashboard.
Perdí cambios locales
git reflog — todo commit efímero de los últimos 30 días está ahí.
Repo remoto borrado
GitHub tiene backups internos las primeras 90 días — abrir ticket con soporte.
Commit rompió producción
git revert <hash> + push inmediato.
