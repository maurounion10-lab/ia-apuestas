2026-07-08 🔴 Crítico abierto GitHub PAT expuesto en backup recuperado del Mac
Qué pasaSe encontró un GitHub Personal Access Token (github_pat_[REDACTADO]...) en un archivo .token del backup, y el mismo token embebido en el .git/config de un repo clonado. Cualquiera con acceso a la carpeta podría usarlo.
DóndeDocuments/Claude/Projects/recuperado-mac-20260707/*/outputs/gambeta/.token y outputs/gambeta/ia-apuestas/.git/config.
Causa raízClone hecho con URL que incluye el token (https://x-access-token:PAT@github.com/...). Es la forma que usa Claude para clonar en background, pero el token queda persistido en remote.origin.url.
Propuesta
  1. Rotar el PAT en GitHub ahora (Settings → Developer settings → Personal access tokens → revocar + generar nuevo).
  2. En el clone local nuevo, usar git remote set-url origin https://github.com/maurounion10-lab/ia-apuestas.git sin token.
  3. Autenticar con GitHub CLI (gh auth login) o Credential Manager de Windows.
